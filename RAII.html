Resource Acquisition Is Initialisation

The dark side of CPP
  1. Reference vs pointer in cpp:
    • Reference:
      • Alias for an already existing variable
      • Stores the address of the object that it aliases
      • A reference is a "fixed" pointer to a specific address
      • The compiler will apply dereferencing (*) for you
      • Cannot be reassigned
      • Cannot be NULL, but can be invalid
      • One level of indirection (only reference to a value, not reference to reference)
      • Used in function parameters and return types:
        1. To modify a local variable of the outer scope (borrowing)
      • Created with int i = 4; int& ref = i;
    • Pointer:
      • Variable that holds a memory address
      • Needs to be dereferenced when accessing the value
      • Can be reassigned
      • Can be NULL
      • Any number of levels of indirection
      • Created with int i = 4; int* ref = &i;
  2. RAII:
    • Use unique_ptr<> to automatically call destructors on the heal-allocated objects
    • Use shared_ptr<> to automatically manage shared pointers:
      1. Cleaned up when the last copy goes out of scope
    • The constructor acquires a resource, the destructor releases the resource (always)
