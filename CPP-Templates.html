Template Specialisation

My course is set for an uncharted sea.
  • In many cases we would write a single generic version for all possible data types
  • Template specialisation allows you to override the default implementation for a specific type.
  • Example:
A generic template for a vector would look something like this: 
template <typename T>
class vector
{
    // accessor functions and so forth
    private:
    T* vec_data;   // we'll store the data as block of dynamically allocated
                   // memory
    int length;    // number of elements used
    int vec_size;  // actual size of vec_data
};
While all vectors may be implemented as an array of a given type, a vector of bool may be implemented as an array of integers with each bit corresponding to one entry in the vector. Such an implementation would greatly conserve memory (factor of 32 or 64, depending on the architecture). A boolean implementation would look like this: 
template <>
class vector <bool>
{
    // interface

    private:
    unsigned int *vector_data;
    int length;
    int size;
};
When we decompose this implementation, we have the first line 
template<>
This line indicates that we are working with a full (explicit) specialisation of the template. To complete the full specialisation, we need to follow this line with another one, that states the type for which we are specialising the template. 
class vector<bool>
