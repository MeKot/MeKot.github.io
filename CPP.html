RAII

Optimising code

  1. One memory reference costs about 100 nanoseconds (https://colin-scott.github.io/personal_website/research/interactive_latency.html):
    • Recalculating values can be faster than keeping them in memory
    • Compilers rely on SSA (static single assignment):
      • Each variable is assigned exactly once
      • Every variable is defined before it is used
  2. Steps:
    1. Cleanup:
      • Compiler removes unnecessary memory allocations
      • Compiler keeps as many variables as possible on the stack
    2. Canonicalization:
      • Compiler rewrites control flow to be in a canonical way.
      • Every loop becomes a canonical LLVM loop, and so do if-then-else statements
    3. Abstraction collapsing:
      • Functions, calls and the call graph:
        • Partitioning call graph on clusters of fucntions
        • Leaves are then natural candidates for inlining -> repeated for new leaves
        • Keep in mind things like partial inlining and call graph shuffling
        • Inlining can produce significant optimisations, but can be hard for compiler to get
